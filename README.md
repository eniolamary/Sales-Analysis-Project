# Sales Data Analysis Project (SQL + Power BI)
### Overview
This project explores sales performance, customer behavior, product trends, and regional insights using SQL for data analysis and Power BI for interactive visualization.

### The workflow involved:

- Data cleaning and transformation using SQL
- Exploratory data analysis using SQL queries
- Building an interactive Power BI dashboard for visualization and business insights

### Tools Used
- SQL (data cleaning + analysis)
- Power BI (dashboard & visualization)
- Excel / dataset source

## SQL Work
SQL was used to:

- Clean and prepare raw sales data
- Aggregate revenue, ratings, and customer segments
- Identify top-performing products, regions, and customer groups
- Analyze time-based trends (monthly, quarterly, weekday patterns)
  
## Key Insights from Analysis
- Male customers aged 31–35 purchasing Clothing generated the highest revenue
- The “Other” product category had the highest ratings but lowest sales volume, indicating untapped opportunity
- Fridays consistently recorded the highest daily sales performance
- Sydney was the top-performing region
- Q2 recorded the highest revenue, though overall seasonal variation was minimal

## Business Recommendations
- Increase visibility and marketing for high-rated low-sales products
- Focus on high-value male clothing customers (31–35 segment)
- Run major promotions on Fridays to maximize sales performance
- Replicate successful strategies from Sydney in lower-performing regions
- Maintain stable inventory planning due to consistent demand across quarters

## Power BI Dashboard

The dashboard includes:

- Overview page (KPIs and summary)
- Customer insights analysis
- Product performance breakdown
- Regional and time-based trends
- Final insights & recommendations page
